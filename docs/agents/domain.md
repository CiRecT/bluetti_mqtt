# Domain Docs

This repository uses a single domain context.

## Before exploring

- Read `CONTEXT.md` at the repository root when it exists.
- Read relevant decisions under `docs/adr/`.
- Proceed silently when either location does not exist.

## Language

Use canonical terms from `CONTEXT.md` in issues, specifications, tests, and
implementation. Do not substitute terms that the glossary explicitly avoids.

If required language is missing, reconsider whether it belongs to the domain
or capture the gap through the domain-modeling workflow.

## Decisions

Surface conflicts with existing ADRs explicitly instead of silently overriding
them.
